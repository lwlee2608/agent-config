# Global Instructions

## Core Principles

1. **Separation of Concerns**: Business logic separate from infrastructure
2. **Consistency**: Match existing code patterns and conventions
3. **Read-Only Status**: Status/GET endpoints never modify data
4. **Single Responsibility**: Each component does one thing well
5. **Clean Boundaries**: Clear separation between layers (API, Service, Domain)
6. **Code is Liability**: Keep code as simple and short as possible
7. **Comment Sparingly**: Only comment when doing something weird

## Git Conventions

1. **Commit messages**: Should be descriptive yet concise
